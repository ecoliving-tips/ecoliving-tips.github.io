# Swaram — Project Instructions for Claude Code

## Project Overview
**Swaram** is a static website for Malayalam Christian devotional song chords (guitar & keyboard).
Hosted on **GitHub Pages** at `https://ecoliving-tips.github.io/`.
Zero npm dependencies — the build system uses only Node.js built-ins.

## Tech Stack
- **Static HTML/CSS/JS** — no framework, no bundler
- **Build system**: `node build.js` — generates static pages from markdown + templates
- **Hosting**: GitHub Pages (deploy by pushing to `main`)
- **Backend**: Supabase (song request form only)
- **PWA**: Service worker (`sw.js`) with offline support
- **i18n**: English + Malayalam via `i18n/translations.json`

## Build System (`build.js`)
Run `node build.js` to regenerate everything. It:
1. Reads `songs/index.json` + each `.md` file
2. Generates static song pages at `/songs/<id>/index.html`
3. Generates lyrics-only pages at `/lyrics/<id>/index.html`
4. Generates category landing pages at `/category/<slug>/index.html`
5. Generates artist landing pages at `/artist/<slug>/index.html`
6. Regenerates `sitemap.xml`
7. Pre-renders song cards into `songs.html` (idempotent regex replacement)
8. Updates `sw.js` precache list with all generated URLs

**IMPORTANT**: Always run `node build.js` after modifying songs, templates, or adding pages.

## Song File Format (`songs/*.md`)
```markdown
---
title: Song Title In Malayalam Transliteration
artist: Artist Name
youtube: https://youtube.com/watch?v=VIDEO_ID
category: Holy Mass
key: C
time: 3/4
---

{Intro}
|| C | Am | G | Em ||

{Verse 1}
[C]Lyric text here [Am]more lyrics
[G]Next line of [Em]lyrics

{Chorus}
[F]Chorus lyrics [G]here
```
**Chord notation rules:**
- `{...}` → section label (Verse, Chorus, Intro, Bridge, etc.)
- `|| C | Am | G ||` → chord-only progression line
- `[Chord]text` → chord positioned above the following lyrics
- Empty lines → spacers between sections

## Adding a New Song (Step by Step)
1. Create `songs/<song-id>.md` with frontmatter + chord content
2. Add entry to `songs/index.json` with `id`, `title`, `file`, `artist`, `category`, `key`, `time`
3. Run `node build.js`
4. Verify: check generated files exist in `songs/<id>/`, `lyrics/<id>/`
5. If new category/artist: verify pages in `category/<slug>/`, `artist/<slug>/`

## Template System
Templates use `{{TOKEN}}` placeholders. Partials are injected as `{{HEAD}}`, `{{HEADER}}`, `{{FOOTER}}`, `{{DONATE}}`.
Song-specific: `{{SONG_TITLE}}`, `{{ARTIST}}`, `{{KEY}}`, `{{CHORD_CONTENT}}`, `{{YOUTUBE_EMBED}}`, `{{STRUCTURED_DATA}}`, `{{LYRICS_LINK}}`, etc.

## i18n System
- All translatable elements use `data-i18n="key"` or `data-i18n-placeholder="key"`
- Keys defined in `i18n/translations.json` (en + ml)
- When adding new UI text: add `data-i18n` attribute + add keys to both languages in translations.json

## CSS Conventions (`css/styles.css`)
- Dark theme with CSS variables (`:root`)
- Primary: `#6C3CE1` (purple), Secondary: `#EC4899` (pink), Accent: `#F59E0B` (amber)
- Background: `#0F0B1A`, Cards: `#1A1425`
- Font: Poppins (Google Fonts)
- Print styles at bottom (`@media print`)
- Responsive breakpoints: 768px, 480px

## SEO Checklist
Every page must have: `<title>`, `<meta description>`, `<link rel="canonical">`, OG tags, Twitter Card tags.
Song/lyrics/category/artist pages get Schema.org JSON-LD (auto-generated by build.js).
`sitemap.xml` is auto-regenerated by build.js.

## Key Gotchas
- **Windows line endings**: `parseFrontmatter()` normalizes `\r\n` → `\n`. Keep this in both `build.js` and `js/songs.js`.
- **Idempotent builds**: `build.js` uses regex to replace content between markers. Safe to run repeatedly.
- **No package.json**: This is intentional — zero npm deps. Use only Node.js built-ins in build.js.
- **Generated files are committed**: Since there's no CI, build output must be committed to deploy.
- **sw.js versioning**: Bump the cache version (`swaram-v2`) when making breaking changes to cached resources.
- **Schema.org for traditional songs**: Use `Organization` type (not `Person`) for traditional/institutional artists.
- **Supabase key in request.html**: This is a publishable/anon key — safe to commit.
